# Hum2Tune — Melody-Based Music Recognition from Humming

**FYP Project · Wong Xuan Kai (20495251) · Supervisor: Dr Simon Lau**

Hum2Tune identifies songs from hummed melodies using frequency-based feature extraction and deep learning. The system addresses the *cross-domain gap* between noisy monophonic humming and polyphonic music recordings.

---

## System Architecture

The project implements two parallel recognition paths:

### 1. Hybrid DTW Retrieval *(primary system)*
A two-stage coarse-to-fine retrieval pipeline:
- **Stage 1 (coarse):** Downsampled chroma DTW filters the full database to a shortlist of ~10–20 candidates.
- **Stage 2 (fine):** Pitch contour subsequence DTW + full-resolution chroma DTW re-rank the shortlist.
- Features extracted per query: pYIN pitch → interval contour; HPSS + chroma CENS.
- Weights: vocal pitch (60%), vocal chroma (40%) in `vocal_only` mode.

### 2. CNN-LSTM / Audio Transformer Classifier *(ablation baseline)*
Both models receive 39-channel features (13 MFCC + 13 Δ + 13 ΔΔ) and produce song-class logits. They address RQ2 and RQ3 of the research questions.

| RQ | Question | How addressed |

## Quick Start

```bash
# 1. Install dependencies
pip install -r requirements.txt

# 2. Prepare the dataset  (place audio in data/Humming Audio/<song_name>/)
python scripts/prepare_dataset.py

# 3a. Train CNN-LSTM with curriculum learning
python scripts/train_model.py --model cnn_lstm

# 3b. Train Audio Transformer
python scripts/train_model.py --model audio_transformer

# 4. Evaluate
python scripts/evaluate.py --model cnn_lstm
python scripts/evaluate.py --model audio_transformer

# 5. Predict on a single hum
python scripts/predict.py --audio path/to/hum.wav --model cnn_lstm

# 6. Run the Streamlit app (hybrid DTW retrieval)
streamlit run src/fyp_title11/app.py
```

## Feature Extraction

```
n_mfcc = 13   →   MFCC (13) + Δ-MFCC (13) + ΔΔ-MFCC (13) = 39 channels
Filterbank: fmin=65 Hz (C2) — fmax=2093 Hz (C7)   ← vocal frequency range
hop_length = 512   sample_rate = 22050
```

All models use `input_channels=39`. The YAML config, model defaults, and evaluate/predict scripts are kept consistent automatically.

## Evaluation Metrics

| Metric | Description |
|--------|-------------|
| Top-1 / Top-3 / Top-5 | Fraction of queries where correct song appears in top K |
| MRR | Mean Reciprocal Rank |
| MAP@10 | Mean Average Precision at rank 10 |
| NDCG@10 | Normalised Discounted Cumulative Gain at rank 10 |
| Macro F1 | Per-class classification performance |

## What is NOT in this repository

By design, the following are excluded (see `.gitignore`):
- Raw audio files (`data/raw/`, `data/hummed/`, `data/processed/`)
- Trained model weights (`models/**/*.pth`)
- Generated logs and result plots
- Reference feature caches (`outputs/reference_feature_cache/`)

Run `prepare_dataset.py` and `train_model.py` to regenerate all artifacts.
